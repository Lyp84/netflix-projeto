criação da função: fc_formatar_duracao_com_segundos_filmes;
Esta função recebe um número inteiro que representa duração em segundos e retorna uma string formatada no formato "HH:MM:SS";

define as variaveis:
    horas tipop: inteiro
    minutos tipo: inteiro
    segundos tipo: inteiro

Lógica interna:
floor(duracao_segundos / 3600): divide o total de segundos por 3600 (quantos segundos tem uma hora) e usa a função "floor" para pegar apenas a parte inteira, arredondando para baixo; 
duracao_segundos % 60: pega o resto da divisão por 60, que são os segundos que não completam um minuto;
lpad(horas::text, 2, '0'): converte o número de horas para texto e usa "lpad" para preencher com zeros à esquerda até ter 2 caracteres;
concat: junta as horas, minutos e segundos formatados com ":" entre eles,adiciona zero à esquerda se necessário;

Criação da função: fc_processar_acao_usuario;
Esta função é executada automaticamente por um trigger quando uma nova ação é inserida na tabela acao_usuario;
Parâmetro: NEW (representa a nova linha inserida na tabela);
Funcionamento por tipo de ação:

Ação 'iniciar':
Verifica se já existe um histórico ativo (sem data de fim) para esse perfil e conteúdo;
Se existir, atualiza a data de início;
Se não existir, cria um novo registro no histórico com porcentagem 0%;

Ação 'pausar':
Busca o histórico ativo mais recente;
Se encontrado e tiver porcentagem informada, atualiza a porcentagem assistida;

Ação 'concluir':
Busca o histórico ativo;
Marca como concluído: data de fim = hora atual, porcentagem = 100%;
Chama função para atualizar preferências do perfil;

Ação 'parar':
Busca o histórico ativo;
Se não informou porcentagem, calcula automaticamente baseado no tempo assistido;
Atualiza histórico com data de fim e porcentagem;
Se assistiu mais de 50%, atualiza preferências do perfil;
Mostra uma mensagem informando a porcentagem assistida e o ID do histórico;

Criação do TRIGGER:
Cria um gatilho chamado trg_processar_acao;
Configuração: executa depois de inserir na tabela acao_usuario;
Para cada linha (FOR EACH ROW);
Executa a função fc_processar_acao_usuario;


Criação da função: fc_atualizar_preferencias_perfil;
Esta função recalcula as preferências de gênero de um perfil baseado no seu histórico de visualização;
Funcionamento interno:

Primeiro limpa todas as preferências antigas do perfil;
Depois recalcula baseado no histórico:
Considera apenas conteúdos com mais de 50% assistidos;
Calcula score para cada gênero: (quantidade de vezes assistiu * avaliação média * fator de recência);
Fórmula: (contagem de visualizações / total assistido) × (avaliação média / 5) × (1.2 se recente, 0.8 se antigo);
Insere na tabela preferencia_perfil;

arquivo 3_validação_classificação

Criação da função: fc_validar_classificacao_perfil;
Parâmetros: p_perfil_id (ID do perfil), p_conteudo_id (ID do conteúdo)
Funcionamento: 

Criação da função: fc_validar_classificacao_perfil;
Esta função verifica se um perfil tem idade suficiente para assistir um determinado conteúdo baseado na classificação indicativa;
Recebe dois parâmetros: p_perfil_id (ID do perfil) e p_conteudo_id (ID do conteúdo);
Retorna verdadeiro (true) se o perfil pode assistir, falso (false) se não pode;

Funcionamento interno passo a passo:

Busca dados do perfil:
Faz uma consulta na tabela perfil para pegar a idade (idade) e se é perfil infantil (is_infantil);
Usa SELECT ... INTO para armazenar esses valores nas variáveis v_idade_perfil e v_is_infantil;
Se não encontrar o perfil (idade é nula), retorna falso;

Busca classificação do conteúdo:
Faz uma consulta na tabela conteudo para pegar a classificação indicativa do conteúdo;
Usa conteudo.classificacao_indicativa%type para garantir que a variável v_classificacao tenha exatamente o mesmo tipo da coluna na tabela;
Bloqueio especial para perfis infantis:

Se o perfil é marcado como infantil (v_is_infantil = true):
Só permite conteúdo classificado como "L" (Livre);
Retorna verdadeiro apenas se a classificação for "L", falso para qualquer outra;

Mapeamento de idade mínima:
Usa a estrutura CASE(verificação de cada linha, quando possui o valor especificado em uma das linhas exeuta ação x) para converter a classificação em idade mínima:
'L' = 0 anos
'10+' = 10 anos
'12+' = 12 anos
'14+' = 14 anos
'16+' = 16 anos
'18+' = 18 anos
Qualquer outra = 18 anos

Comparação final:
Retorna verdadeiro se a idade do perfil (v_idade_perfil) for maior ou igual à idade mínima requerida (v_min_idade);
Retorna falso se a idade for menor;


criação trigger:
Cria um gatilho chamado trg_validar_classificacao;
Configuração: executa depois de inserir na tabela historico;
Para cada linha (FOR EACH ROW);
Executa a função fc_bloquear_conteudo_inadequado;

criação trigger:
Cria um gatilho chamado trg_validar_classificacao_update;
Configuração: executa depois de atualizar a tabela historico;
Para cada linha (FOR EACH ROW)
quando novo conteudo_id for diferente de antigo conteudo_id;
Executa a função fc_bloquear_conteudo_inadequado;
